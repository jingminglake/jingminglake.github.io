## 蓝图卡片

### 卡片：可变滑动窗口（核心蓝图）

**根本前提**：单调性 — 扩大窗口合法性只变差，缩小窗口合法性只变好。新题第一步先验证这个，不满足就不能用。

**三个核心问题**：
1. **窗口内维护什么？** 由"合法"的定义决定（计数器 / hash map / 单调队列）
2. **right 进攻** — 走到位置 i 时，left 之前所有位置与 i 的组合已全部处理，不遗漏
3. **left 防守** — 触发时机和目标由求最大还是求最小决定（见下方对比）

**求最大 vs 求最小 — left 的触发时机正好相反**：

| | 求最大窗口（如 1004） | 求最小窗口（如 LC 76） |
|--|---------------------|----------------------|
| left 移动时机 | 窗口**不合法**时 | 窗口**合法**时 |
| left 停止条件 | 停到刚好合法 | 停到刚好不合法 |
| left 的目标 | 找第一个合法位置 | 找最后一个合法位置（沿途记录答案） |

**复杂度**：摊还分析 — 单次 while 可能多步，但 left/right 全程只增不减，总计 O(2N)。

### 卡片：1004 — 至多 k 个 0

- 维护：`zeros`（窗口内 0 的个数）
- 不合法：`zeros > k`
- 收缩：while，每次 right 最多引入 1 个 0，但消除它可能要扫过多个 1

### 卡片：Leetcode 3 — 无重复字符

- 维护：hash map（字符 → 最后出现下标）
- 不合法：新字符的 `last_seen[c] >= left`（在窗口内才算重复）
- 收缩：**不需要 while**，`left = last_seen[c] + 1` 一跳到位
- 为什么一跳到位：每次 right 只引入一个字符，窗口内最多一个旧副本

### 卡片对比：while vs 一跳到位

| | 1004 | LC 3 |
|--|------|------|
| 每次 right 引入几个违规 | 最多 1 个 0 | 最多 1 个重复 |
| 消除违规需要几步 | 不定（要扫过 1）→ while | 恰好 1 步（位置已知）→ if |
